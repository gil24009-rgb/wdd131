NPS Park Explorer

A simplified multi page park information experience for Yellowstone using the National Park Service API.

Project type

Multi page informational website

Primary park

Yellowstone, park code yell

Core capability

Live API driven rendering

Purpose

The purpose of NPS Park Explorer is to provide a clear, fast overview of Yellowstone National Park. It prioritizes information that visitors need right before or during a trip: park intro, alerts, visitor services, and activities. Data is fetched from the official NPS API so content stays current.

Primary goals

  • Reduce time to find critical information
  • Highlight safety and closure alerts
  • Use expandable sections to prevent mobile overwhelm
  • Keep content current through live API data

Audience

This site is designed for visitors who want essential park info without scanning a large official site.

Primary audience

  • First time visitors planning a trip
  • Families and groups checking current conditions
  • Mobile users on the go

Secondary audience

  • Students and instructors reviewing API driven work
  • Anyone needing quick alert visibility

Dynamic elements description

JavaScript is used to fetch live data from the NPS API and render content dynamically. The codebase is modular to separate data access, templates, and page behavior.

Data driven rendering

  • Fetch park details from the parks endpoint with async and await
  • Fetch alerts from the alerts endpoint using parkCode
  • Fetch visitor centers from the visitorcenters endpoint using parkCode
  • Render lists using map and join

Interactive behaviors

  • Mobile global menu toggle with aria-expanded updates
  • Accordion sections using details for progressive enhancement
  • Alert icon mapping based on alert category
  • Production safe asset paths for SVG sprite usage

Logo

The logo uses a shield inspired base and a simple landscape shape to communicate parks and exploration. It is designed to remain readable at small sizes and to work on dark headers.
