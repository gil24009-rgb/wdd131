Purpose

The purpose of NPS Park Explorer is to provide a clean, easy to scan overview of a national park. It focuses on the information a visitor needs right before or during a trip: key park intro, quick navigation, alerts, visitor services, and activities. Data is pulled from the official NPS API to keep the content current.

Primary goals

  • Provide a fast overview of the park with clear hierarchy and readable layout
  • Surface time sensitive alerts in a way that is easy to notice and understand
  • Offer expandable sections to reduce cognitive load on mobile screens
  • Ensure the site works in production deployment and not only in development

Audience

This site is designed for visitors who want essential park information without having to scan a large official site.

Primary audience

  • First time park visitors planning a trip
  • Families and groups checking current conditions
  • Visitors already at the park who need alerts and visitor center details

Secondary audience

  • Students and instructors reviewing API driven front end work
  • Users on mobile devices with limited time and limited screen space

Dynamic elements description

JavaScript will be used to fetch live data from the NPS API and update the page content dynamically. The site uses a modular approach to separate data access, templates, and page specific behaviors.

Data driven rendering

  • Fetch park details from the parks endpoint using async and await
  • Fetch alerts from the alerts endpoint using the park code
  • Fetch visitor centers from the visitorcenters endpoint using the park code
  • Render templates into the DOM using map and join

Interactive behaviors

  • Mobile global menu toggle with aria expanded state updates
  • Accordion style sections using details for progressive enhancement
  • SVG icon selection based on alert category with a category mapping rule
  • Path handling for production by using base url aware asset references

Example dynamic flow

On page load, the app fetches park data first. It then uses the park code to request alerts and visitor centers. The results are inserted into lists using templates. This ensures content remains current without manual updates.

Logo

The logo uses a simple shield inspired shape and a minimal landscape mark to communicate parks and exploration. It is designed to be readable at small sizes and to work on dark and light backgrounds.

In production the logo can be exported as a standalone SVG file if needed.

Content

This section includes the planned text content for each page. Final content is populated from the NPS API where available.

Home page content

  • Hero title: Yellowstone National Park
  • Intro paragraph: A short summary of the park from the API description field
  • Info cards: Current conditions, Fees and passes, Visitor centers
  • Footer: Address and phone from API contacts data

Sample text

Yellowstone is a place for geysers, wildlife, and wide open landscapes. Use this site to quickly check alerts, visitor services, and activities before you go.

Child page content, Current Conditions

  • Section: Alerts with category icon, title, and description
  • Section: Visitor services with visitor center list, description, and directions
  • Section: Activities list from the park data activities array
  • Behavior: Accordion style sections for visitor centers and activities

Sample text

Review alerts before you travel, then expand the visitor services section to find visitor center details. Browse activities to plan your day.
